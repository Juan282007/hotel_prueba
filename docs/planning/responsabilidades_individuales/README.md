# Guias individuales de responsabilidades

Esta carpeta tiene una guia separada para cada integrante. Cada archivo explica:

- HUs asignadas.
- Rama exacta que debe crear desde `dev`.
- Rama hija exacta que debe crear desde `qa`.
- Rama hija exacta que debe crear desde `main`.
- Archivos que debe subir.
- `author` de Liquibase cuando aplica.
- Quien aprueba cada PR.
- Quien hace merge en cada etapa.
- Cuando debe bajar cambios con `git pull`.

Antes de trabajar, cada integrante debe leer tambien:

```text
docs/entrega-base-datos/que_subir_y_que_no_subir.md
```
